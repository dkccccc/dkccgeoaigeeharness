# datawa_study05 · 여러 지역 자동 분석 웹앱

5장의 결과물. [4장 변화탐지 서비스 웹앱](../datawa_study04/)에 **여러 지역 일괄 분석** 모드를 더한 **누적 Streamlit 웹앱**입니다.
4장까지는 지역 한 곳을 손으로 한 번씩 돌렸습니다. 5장은 앱 상단에서 모드를 고르면, **여러 지역**의 이름·좌표 목록만 주고 **[자동 분석]** 한 번이면 전부 분석해 **한 표로 모아줍니다** — 자동화의 출발점입니다.

이 한 파일(`app.py`)이 0~5장 누적 앱 전체입니다. 분석 로직(`run_change_detection`)은 4장 그대로이고, 그 위에 모드 선택과 배치 분석만 얹었습니다.

## 만든 방법 (뼈대 → 기능 → 점검)

[PROMPTS.md](../PROMPTS.md)의 프롬프트 **5-1 → 5-2 → 5-3** 순서로 Claude Code에게 시켜 단계별로 키웠습니다.

- **5-1 뼈대** — 앱 상단에 모드 선택을 두고, `이름,위도,경도` 목록을 받아 4장 `run_change_detection` 을 **지역마다 차례로 호출**하는 `batch_change_detection` 부터. 한 지역이 실패해도 멈추지 않습니다. 표는 아직.
- **5-2 기능** — 모인 결과를 지역명·평균변화·감소%·증가% **결과 표(`st.dataframe`)** 로 묶고(`build_report`), "가장 주의해서 볼 지역" 한 줄 요약과 CSV 내려받기를 붙입니다.
- **5-3 점검** — 표의 숫자가 상식 범위인지 **교차 확인**합니다(평균 ±1 초과, 감소%+증가% 100% 초과 의심). 가장 많이 감소한 지역의 좌표를 지도에서 직접 봅니다.

## 혼자(loop) vs 팀(에이전트 팀)

`batch_change_detection` 은 **혼자 차례로 도는 loop 방식**입니다 — 가장 단순하고 확실합니다.
같은 일을 **Claude Code에게 "여러 지역을 나눠 동시에 맡는 작은 팀처럼"** 시킬 수도 있습니다(에이전트 팀). 지역이 수십 곳으로 늘면 그쪽이 빨라집니다. **이 교재 자체가 5인 에이전트 팀이 나눠 만든 결과물**입니다(저장소 `.claude/`).

## 설치

```bash
pip install -r requirements.txt
```

(4장 환경이 있다면 `pandas` 만 추가로 깔면 됩니다.)

## 실행

먼저 Earth Engine 인증을 **터미널에서 한 번** 해 둡니다. (웹앱은 브라우저 인증 창을 띄울 수 없어서, 미리 토큰을 만들어 둡니다.)

```bash
earthengine authenticate     # 처음 한 번 (브라우저가 열리며 로그인)
streamlit run app.py         # 앱 실행
```

실행하면 터미널에 `Local URL: http://localhost:8501` 이 뜨고, 브라우저가 자동으로 열립니다.

## 사용법

앱 상단에서 **모드**를 고릅니다.

**“한 지역”** (4장 그대로)
1. 왼쪽 사이드바에서 중심 위도·경도·반경(km)으로 영역을 정한다.
2. 이전/이후 기간과 구름 임계값을 고른다.
3. **[② 분석 실행]** → 요약 카드 + 변화 지도 + JSON 내려받기.

**“여러 지역”** (이 장의 핵심)
1. 가운데 텍스트 영역에 `이름,위도,경도` 를 **한 줄에 하나씩** 붙여넣는다. (구글지도 우클릭하면 `위도, 경도` 가 보입니다.)
   ```
   서울숲,37.5444,127.0375
   세종신도시,36.4800,127.2890
   송도,37.3830,126.6490
   ```
2. 사이드바에서 공통 조건(반경·기간·구름 임계값)을 정한다 — 모든 지역에 같이 적용됩니다.
3. **[② 여러 지역 자동 분석]** → 진행 막대가 지역마다 한 칸씩 차고, 끝나면 **결과 표** + "가장 주의해서 볼 지역" 요약 + CSV 내려받기가 나온다.
4. 한 지역이 실패(맑은 영상 없음 등)해도 멈추지 않고 나머지를 계속 분석합니다. 건너뛴 지역은 표 아래 펼침에서 이유와 함께 봅니다.

## 주요 함수

- `run_change_detection(roi, before, after, cloud_pct=20)` → **한 지역**을 분석해 통계 dict(`mean_change`, `decrease_ratio`, `increase_ratio`, `area_km2`, `image`, `roi` …) 반환. **4장 그대로** — 5장은 이 함수를 여러 지역에 반복 호출만 합니다(경계면 일치).
- `make_roi(lon, lat, radius_km)` → 중심 좌표와 반경으로 네모 영역(`ee.Geometry`)을 만든다. (4장 함수)
- `parse_aoi_text(text)` → 텍스트 영역의 `이름,위도,경도` 줄들을 `[{"name","lon","lat"}, ...]` 목록으로 바꾼다. 빈 줄·형식 오류 줄은 건너뜀.
- `batch_change_detection(aoi_list, before, after, cloud_pct=20, radius_km=3, progress=None)` → **여러 지역을 차례로 분석**해 지역별 통계 dict 목록을 반환. 한 지역이 실패해도 멈추지 않고 `"error"` 를 담아 다음으로 넘어간다.
- `build_report(results)` → 결과 목록을 **표(`pandas.DataFrame`)** + "가장 주의해서 볼 지역" 요약 + 성공/실패 수로 묶는다. 가장 많이 감소한 지역이 맨 위.

## 확인 (이게 보이면 성공)

- 앱 상단에 **“한 지역 / 여러 지역”** 모드 선택이 보인다.
- “여러 지역” 모드에서 **[자동 분석]** 을 누르면 진행 막대가 지역마다 한 칸씩 찬다.
- 끝나면 지역명·평균변화·감소%·증가% **표**가 뜨고, 가장 많이 감소한 지역이 **맨 위**에 온다.
- 표 아래 **"가장 주의해서 볼 지역: …"** 한 줄 요약과 **CSV 내려받기** 버튼이 보인다.
- 한 지역이 실패해도 나머지 지역의 행은 표에 그대로 남는다.

## 막히면

- **지도가 빈 회색** → `st_folium(m)` 으로 그렸는지, GEE 레이어를 `add_ee_layer`(getMapId)로 올렸는지 확인.
- **한 지역만 "건너뜀"** → 그 기간에 맑은 영상이 없는 것. 사이드바의 `구름 임계값` 을 30~40으로 올리거나 기간을 넓히세요. 나머지 지역은 정상 처리됩니다.
- **모든 지역 "건너뜀"** → 인증/초기화가 안 됐거나 좌표 형식이 틀린 것. 터미널에서 `earthengine authenticate` 를 먼저 하고, 목록을 `이름,위도,경도`(쉼표·숫자) 형식으로 적었는지 확인하세요.
- **목록이 비었다는 오류** → 텍스트 영역에 `이름,위도,경도` 가 한 줄도 없는 것. 한 줄에 하나씩, 칸을 쉼표로 나눠 적으세요. (위도가 앞, 경도가 뒤)
- **평균 변화량이 ±1을 넘게 큼** → `before`/`after` 가 계절이 다른 영상일 수 있음. 두 기간을 같은 계절로 맞추세요.
- **느림** → 지역 수가 많거나 반경이 큰 것. 지역을 줄이거나 사이드바 `각 지역 반경` 을 2~3km로 줄이세요.
- **`streamlit: command not found`** → `pip install -r requirements.txt` 후 다시.
- 그 외 에러는 메시지를 복사해 Claude Code에게 물어보세요. (프롬프트 0-2 패턴)

> 코드 검증 메모: `run_change_detection` 의 시그니처·반환 dict(`mean_change`, `decrease_ratio`, `increase_ratio`, `area_km2`)는 4장 `datawa_study04/service.py`·6장 `datawa_study06/app.py` 와 동일합니다. 5장은 이 함수를 여러 지역에 반복 호출만 합니다(경계면 일치). Streamlit 안 EE 지도는 `ee.getMapId()` 타일을 folium에 올리고 `st_folium()` 으로 그립니다 (dkidi와 같은 raw ee + Leaflet 방식).
