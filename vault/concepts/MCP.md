# MCP

> [!question] 왜 필요한가
> AI에게 "이 위성영상을 지도 서비스에 올려줘"라고 하고 싶다. 그런데 AI는 내 지도 서비스, 내 데이터베이스, 내 파일에 어떻게 손을 뻗지? 매번 복사해서 붙여넣어 줘야 하나?

## 한마디로
**AI가 바깥 도구·데이터에 연결되는 표준 통로.** AI를 위한 **USB 포트**라고 보면 된다. 예전엔 기기마다 케이블이 다 달랐지만, USB라는 약속이 생기자 아무 기기나 같은 구멍에 꽂으면 됐다. MCP는 AI 세계의 그 "공통 구멍"이다.

## 조금 더
MCP는 **M**odel **C**ontext **P**rotocol의 줄임말이다. 풀어 쓰면:
- **Model** — AI 모델 (예: Claude)
- **Context** — AI가 일하는 데 필요한 바깥 정보·도구
- **Protocol** — 서로 연결하는 약속(표준)

MCP가 있으면 AI는 자기 머릿속 지식만 쓰는 대신, **바깥의 진짜 도구와 데이터에 직접 닿는다** — 지도 서비스, 데이터베이스, 파일 저장소 같은 것. AI가 "혼자 생각만 하는 비서"에서 "직접 도구를 쥐고 일하는 비서"로 바뀌는 셈이다.

> [!note] 왜 "표준"이 중요한가
> 도구마다 연결법이 다르면 새 도구를 붙일 때마다 처음부터 다시 만들어야 한다. MCP라는 공통 약속을 따르면, 약속을 지키는 도구는 무엇이든 같은 방식으로 꽂힌다. 그래서 [[deployment|배포]]한 서비스에 새 기능을 붙이기가 훨씬 쉬워진다.

> [!tip] 우리 프로젝트에서는
> 6장에서 변화탐지 서비스에 MCP로 외부 도구를 연결해, "두 시점 비교"에 머물지 않고 기능을 확장하는 길을 살짝 맛본다. 깊은 설정은 도구가 처리하고, 여러분은 "무엇을 연결하면 좋을지"를 고민하면 된다.
