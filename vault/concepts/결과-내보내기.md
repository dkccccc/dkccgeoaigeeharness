# 결과 내보내기 — 분석 결과를 파일로 저장하기

> [!question] 왜 필요한가
> 화면에서 변화탐지 결과를 멋지게 봤다. 그런데 동료에게 보내거나 QGIS·엑셀에서 더 분석하려면? 화면은 닫으면 사라진다. 결과를 **파일로 저장**해야 남기고 나눌 수 있다.

## 한마디로
**결과 내보내기** = 화면에 그려진 분석 결과를 다른 프로그램이 읽을 수 있는 파일로 저장하는 일.
- **스크린샷**은 보기만 되는 그림이고,
- **내보낸 파일**은 다른 사람이 열어서 다시 편집·분석할 수 있는 원본이다.

우리가 주로 쓰는 두 가지: **GeoJSON**과 **CSV**.

## 조금 더
- **GeoJSON** = 지도용 파일. 각 결과에 **위치(좌표)**가 붙어 있어서 QGIS 같은 지도 프로그램에 넣으면 그 자리에 바로 표시된다. "어디서 변했나"를 지도로 보고 싶을 때.
  - JSON은 컴퓨터가 읽기 쉽게 정리한 텍스트 형식이고, GeoJSON은 거기에 지리 위치를 담은 것이다.
- **CSV** = 표 파일. 쉼표로 칸을 나눈 단순한 표로, **엑셀**에서 바로 열린다. "변화 면적이 얼마", "픽셀이 몇 개" 같은 **숫자·통계**를 정리할 때.

> [!note] 어느 걸 고를까
> 지도에 다시 띄울 거면 GeoJSON, 표로 계산·집계할 거면 CSV. 둘 다 내보내도 된다 — 같은 결과를 다른 그릇에 담는 것뿐이다.

> [!tip] 우리 프로젝트에서는
> 4장에서 [[변화탐지란|변화탐지]] 결과를 파일로 내보낸다. 변한 지역을 GeoJSON으로 저장해 동료가 QGIS에서 확인하게 하고, 변화 면적 같은 수치는 CSV로 내보내 엑셀에서 추가 분석한다. 이렇게 "쓸 만한 서비스"는 화면 너머로 결과를 전달한다.
