# 4장 · 쓸 만한 변화탐지 서비스

> [!success] 이 장의 결과물
> 기간·지역·구름 임계값을 **변수만 바꿔** 다른 곳에도 돌릴 수 있는 변화탐지 서비스, 그리고 그 결과를 **GeoJSON·CSV 파일로 내보내** 동료에게 보내고 엑셀·QGIS로 다시 열 수 있는 상태. 화면 너머로 결과를 전달하는 "쓸 만한 서비스".

[[chapters/3-change-detection|3장]]까지는 "내가 그린 한 영역, 정해둔 한 기간"을 한 번 보여주는 코드였습니다. 이 장은 같은 변화탐지를 **남이 가져다 쓸 수 있는 서비스**로 바꾸고, 그 결과를 **파일로 떨어뜨려** 가져갈 수 있게 합니다.

## 이론

### 결과 내보내기 (GeoJSON·CSV)

화면에서 변화탐지 결과를 멋지게 봤습니다. 그런데 동료에게 보내거나 QGIS·엑셀에서 더 분석하려면? **화면은 닫으면 사라집니다.** 결과를 **파일로 저장**해야 남기고 나눌 수 있습니다.

**결과 내보내기**란 화면에 그려진 분석 결과를 다른 프로그램이 읽을 수 있는 파일로 저장하는 일입니다. **스크린샷**은 보기만 되는 그림이지만, **내보낸 파일**은 다른 사람이 열어서 다시 편집·분석할 수 있는 원본입니다. 우리가 주로 쓰는 두 가지는 **GeoJSON**과 **CSV**입니다.

- **GeoJSON = 지도용 파일.** 각 결과에 **위치(좌표)**가 붙어 있어서 QGIS 같은 지도 프로그램에 넣으면 그 자리에 바로 표시됩니다. "어디서 변했나"를 지도로 보고 싶을 때 씁니다. (JSON은 컴퓨터가 읽기 쉽게 정리한 텍스트 형식이고, GeoJSON은 거기에 지리 위치를 담은 것입니다.)
- **CSV = 표 파일.** 쉼표로 칸을 나눈 단순한 표로, **엑셀**에서 바로 열립니다. "변화 면적이 얼마", "픽셀이 몇 개" 같은 **숫자·통계**를 정리할 때 씁니다.

> [!note] 어느 걸 고를까
> 지도에 다시 띄울 거면 GeoJSON, 표로 계산·집계할 거면 CSV. 둘 다 내보내도 됩니다 — 같은 결과를 다른 그릇에 담는 것뿐입니다.

이 장에서는 [[chapters/3-change-detection|변화탐지]] 결과를 파일로 내보냅니다. 변한 지역을 GeoJSON으로 저장해 동료가 QGIS에서 확인하게 하고, 변화 면적 같은 수치는 CSV로 내보내 엑셀에서 추가 분석합니다. 이렇게 "쓸 만한 서비스"는 화면 너머로 결과를 전달합니다.

## 실습 — 변화탐지 서비스

> [!info] 이 실습을 마치면
> 기간·지역·구름 임계값을 **변수만 바꿔** 다른 곳에도 돌릴 수 있는 변화탐지 서비스가 생기고, 그 결과를 **GeoJSON/CSV 파일로 내보내** 엑셀·QGIS로 열 수 있습니다. 약 35분.
> **선행:** [[chapters/3-change-detection|3장]] 의 변화탐지(`ndvi_change`)가 동작해야 합니다 (두 시점 NDVI를 빼서 변화 이미지를 얻을 수 있어야 함).

리듬은 그대로 **뼈대 → 기능 → 점검**입니다. ([[chapters/1-prompts|프롬프트 4단계]])

### 단계 1 — 뼈대 (프롬프트 4-1)

[[chapters/3-change-detection|3장]] 변화탐지를 **함수 하나로 감싸**, 지역·기간·구름 임계값을 인자로 받게 만듭니다. 아직 파일 내보내기는 신경 쓰지 않습니다 — **변수만 바꿔 다시 돌아가는지**부터 확인합니다.

> [!quote] 4-1
> 좋아, 3장 변화탐지가 잘 돌아가. 이제 이걸 '서비스'로 만들고 싶어. 지역, 이전 기간, 이후 기간, 그리고 구름 임계값을 인자로 받는 `run_change_detection` 함수 하나로 감싸줘. 변화 이미지 하나만 주지 말고, 평균 변화량과 식생이 감소·증가한 면적 비율 같은 요약 통계도 같이 담은 딕셔너리로 돌려줘. 그래야 나중에 여러 지역에 반복해 쓸 수 있어.

왜 이미지가 아니라 dict(통계까지)로 돌려줄까요? → 위 [이론](#결과-내보내기-geojsoncsv)에서 다룬 것처럼, 화면에 띄울 이미지 말고 **파일로 옮길 숫자**까지 손에 들고 있어야 나중에 내보낼 수 있습니다. 구름 임계값을 왜 인자로 빼나요? → [[chapters/1-prompts|구름 필터]]

> [!success] 확인
> ```python
> roi    = m.user_roi
> before = ("2023-01-01", "2023-12-31")
> after  = ("2024-01-01", "2024-12-31")
> result = run_change_detection(roi, before, after, cloud_pct=20)
> ```
> 를 실행하면 에러 없이 아래처럼 요약이 찍힙니다. `before`/`after`/`cloud_pct` 만 바꿔도 다시 돌아가면 성공입니다.
> ```
> [변화탐지 완료] 영역 12.4 km^2
>   평균 변화량 : -0.018  (+증가 / -감소)
>   식생 감소 면적 : 8.3 %
>   식생 증가 면적 : 4.1 %
> ```

### 단계 2 — 기능 (프롬프트 4-2)

이제 그 결과를 **파일로 내보냅니다.** 영역을 작은 격자(셀)로 나눠 셀별 평균 변화량을 계산하고, **GeoJSON/CSV**로 저장해 다운로드할 수 있게 합니다.

> [!quote] 4-2
> 이제 이 결과를 파일로 내보내고 싶어. 영역을 2km 정도 격자로 나눠서 각 칸의 평균 변화량을 계산하고, GeoJSON과 CSV 둘 중에 골라서 저장하는 `export_result` 함수를 만들어줘. CSV는 엑셀로, GeoJSON은 QGIS로 바로 열 수 있게. 파일 이름은 기간이 들어가게 자동으로 지어줘.

격자(셀) 단위로 내보내면 뭐가 좋을까요? → 영역 전체를 한 덩어리 숫자 하나로 뭉개지 않고, **칸칸이 어디가 얼마나 변했는지**를 남깁니다. 그래야 GeoJSON으로 지도에 펴 봤을 때 "이 칸이 가장 많이 줄었다"가 그 자리에 보입니다.

> [!success] 확인
> ```python
> path = export_result(result, fmt="csv")   # 또는 fmt="geojson"
> ```
> 를 실행하면 `change_20230101_to_20240101.csv` 같은 파일이 생기고, 아래처럼 안내가 찍힙니다.
> ```
> CSV 로 저장했어요 → change_20230101_to_20240101.csv  (엑셀로 바로 열 수 있어요)
> ```
> 노트북을 코랩에서 돌렸다면 왼쪽 파일 탭에, 로컬이면 작업 폴더에 그 파일이 떨어져 있으면 성공입니다.

### 단계 3 — 점검 (프롬프트 4-3)

파일이 만들어졌다고 끝이 아닙니다. **그 파일을 다시 열어** 값이 상식과 맞는지 봅니다. 셀 개수가 영역 크기와 어울리는지, 변화량이 터무니없이 크지 않은지(±1을 넘으면 구름·계절 노이즈 의심) 확인합니다. 코드가 만든 숫자를 그대로 믿지 않는 습관입니다. ([[chapters/0-first-vibe-coding|할루시네이션 검증]])

> [!quote] 4-3
> 방금 내보낸 파일을 다시 열어서 값이 제대로 들어갔는지 확인하고 싶어. 셀이 몇 개고, 변화량의 최소·최대·평균이 얼마인지 출력해줘. 변화량이 보통 어느 범위면 정상인지도 알려줘. 너무 크면 뭘 의심해야 해?

> [!success] 확인
> ```python
> check_export(path)
> ```
> 를 실행하면 셀 개수와 변화량 범위가 찍힙니다. 범위가 대략 **-0.5 ~ +0.5** 면 정상입니다.
> ```
> 파일 확인 → change_20230101_to_20240101.csv
>   셀 개수      : 9 개
>   변화량 최소  : -0.214  (가장 많이 감소한 셀)
>   변화량 최대  : +0.157  (가장 많이 증가한 셀)
>   변화량 평균  : -0.018
> ```
> CSV를 엑셀로 직접 열어, 가장 많이 감소한 셀의 좌표가 실제 개발지·벌채지 근처인지도 눈으로 대보세요. ([[chapters/0-first-vibe-coding|할루시네이션 검증]])

### 막히면

> [!warning] 자주 나는 오류
> - **`roi` 가 None** → 지도에서 영역을 먼저 그린 뒤 셀을 다시 실행. ([[chapters/3-change-detection|3장]])
> - **`cell_change` 가 전부 빈칸** → 그 기간에 맑은 영상이 없는 것. `cloud_pct` 를 30~40으로 올리거나 기간을 넓히세요. → [[chapters/1-prompts|구름 필터]]
> - **변화량이 ±1을 넘게 큼** → `before` 와 `after` 가 계절이 다른 영상일 수 있음. 두 기간을 같은 계절(예: 둘 다 여름)로 맞추세요. → [[chapters/3-change-detection|변화 해석하기]]
> - **셀이 수백 개로 폭증 / 느림** → 영역이 큰 것. `export_result(result, grid_km=5)` 처럼 격자를 키워 셀 수를 줄이세요.
> - **`ee_to_geojson` 없다는 에러** → geemap 버전이 낮은 것. `pip install -U geemap` 로 올려보세요.
> - 완성 코드는 `datawa_study04/service.py` 참고.

## 다음
