# 4장 · 쓸 만한 변화탐지 서비스

> [!success] 이 장의 결과물
> 기간·지역·구름 임계값을 **변수만 바꿔** 다른 곳에도 돌릴 수 있는 변화탐지 서비스, 그리고 그 결과를 **GeoJSON·CSV 파일로 내보내** 동료에게 보내고 엑셀·QGIS로 다시 열 수 있는 상태. 화면 너머로 결과를 전달하는 "쓸 만한 서비스".

[[3-change-detection|3장]]까지는 "내가 그린 한 영역, 정해둔 한 기간"을 한 번 보여주는 코드였습니다. 이 장은 같은 변화탐지를 **남이 가져다 쓸 수 있는 서비스**로 바꾸고, 그 결과를 **파일로 떨어뜨려** 가져갈 수 있게 합니다.

## 이론

> [!note]- 결과 내보내기 (GeoJSON·CSV)
> 화면에서 변화탐지 결과를 멋지게 봤습니다. 그런데 동료에게 보내거나 QGIS·엑셀에서 더 분석하려면? **화면은 닫으면 사라집니다.** 결과를 **파일로 저장**해야 남기고 나눌 수 있습니다.
>
> **결과 내보내기**란 화면에 그려진 분석 결과를 다른 프로그램이 읽을 수 있는 파일로 저장하는 일입니다. **스크린샷**은 보기만 되는 그림이지만, **내보낸 파일**은 다른 사람이 열어서 다시 편집·분석할 수 있는 원본입니다. 우리가 주로 쓰는 두 가지는 **GeoJSON**과 **CSV**입니다.
>
> - **GeoJSON = 지도용 파일.** 각 결과에 **위치(좌표)**가 붙어 있어서 QGIS 같은 지도 프로그램에 넣으면 그 자리에 바로 표시됩니다. "어디서 변했나"를 지도로 보고 싶을 때 씁니다. (JSON은 컴퓨터가 읽기 쉽게 정리한 텍스트 형식이고, GeoJSON은 거기에 지리 위치를 담은 것입니다.)
> - **CSV = 표 파일.** 쉼표로 칸을 나눈 단순한 표로, **엑셀**에서 바로 열립니다. "변화 면적이 얼마", "픽셀이 몇 개" 같은 **숫자·통계**를 정리할 때 씁니다.
>
> **어느 걸 고를까**
> 지도에 다시 띄울 거면 GeoJSON, 표로 계산·집계할 거면 CSV. 둘 다 내보내도 됩니다 — 같은 결과를 다른 그릇에 담는 것뿐입니다.
>
> 이 장에서는 [[3-change-detection|변화탐지]] 결과를 파일로 내보냅니다. 변한 지역을 GeoJSON으로 저장해 동료가 QGIS에서 확인하게 하고, 변화 면적 같은 수치는 CSV로 내보내 엑셀에서 추가 분석합니다. 이렇게 "쓸 만한 서비스"는 화면 너머로 결과를 전달합니다.

## 실습 — 쓸 만한 변화탐지 서비스

> [!info] 이 실습을 마치면
> [[3-change-detection|3장]]의 변화탐지 앱에, **구름 임계값 슬라이더**와 **요약 통계 카드**, 그리고 결과를 **GeoJSON/CSV로 내려받는 다운로드 버튼**이 붙어 — 남에게 건넬 수 있는 "쓸 만한 서비스"가 됩니다. 약 35분.
> **선행:** [[3-change-detection|3장]]의 변화탐지 웹앱(`datawa_study03/app.py`)이 `streamlit run app.py` 로 떠야 합니다. 이 장은 그 앱을 이어서 키웁니다.

3장까지의 `app.py`(누적 웹앱)를 그대로 열어 두고, Claude Code에게 **말로** 살을 붙입니다. 리듬은 그대로 **뼈대 → 기능 → 점검**입니다. ([[1-prompts|프롬프트 4단계]])

> [!note] 준비
> 새 폴더가 아니라 **3장 `app.py` 를 복사해** 이어서 키웁니다. 추가 설치는 CSV 표를 위한 pandas 하나뿐입니다.
> ```bash
> pip install pandas        # CSV 표 미리보기용 (나머지는 3장과 동일)
> streamlit run app.py      # 앱을 띄워 둔 채로 아래 프롬프트를 주세요
> ```

### 단계 1 — 뼈대 (프롬프트 4-1): 구름 임계값 + 요약 통계

먼저 3장 변화탐지를 **함수 하나로 감싸** 지역·기간·구름 임계값을 인자로 받게 하고, 사이드바에 **구름 임계값 슬라이더**를, 화면에 **요약 통계 카드**를 띄웁니다. 아직 파일 내보내기는 신경 쓰지 않습니다.

> [!quote] 4-1
> 지금 3장 변화탐지 앱(app.py)이 잘 돌아가. 이걸 '쓸 만한 서비스'로 키우고 싶어.
> 1. 변화탐지를 `run_change_detection(roi, before, after, cloud_pct)` 함수 하나로 감싸줘. 변화 이미지 하나만 주지 말고, **평균 변화량·식생 감소 면적 비율·증가 면적 비율·영역 넓이(km²)** 를 같이 담은 딕셔너리로 돌려줘.
> 2. 사이드바에 **구름 임계값 슬라이더**(cloud_pct, 5~60%)를 추가해서, 그 값으로 영상을 거르게 해줘.
> 3. [분석 / 보기]를 누르면 그 요약 통계를 **st.metric 카드 4개**(평균 변화량·감소%·증가%·면적)로 화면에 띄워줘. 맑은 영상이 없으면 친절히 안내하고 멈추게 해줘.

왜 이미지가 아니라 dict(통계까지)로 돌려줄까요? → 위 [이론](#결과-내보내기-geojsoncsv)처럼, 화면에 띄울 이미지 말고 **파일로 옮길 숫자**까지 손에 들고 있어야 나중에 내보낼 수 있습니다. 구름 임계값을 왜 슬라이더로 빼나요? → [[1-prompts|구름 필터]]

> [!success] 확인 (브라우저)
> 사이드바에 **구름 임계값 슬라이더**가 새로 생기고, [분석 / 보기]를 누르면 지도 위에 카드 4개가 이렇게 뜹니다.
> ```
> 평균 변화량      식생 감소 면적     식생 증가 면적     분석 영역
>   -0.018           8.3 %            4.1 %          12.4 km²
> ```
> 구름 임계값 슬라이더만 30으로 올려도 다시 카드가 갱신되면 성공입니다.

### 단계 2 — 기능 (프롬프트 4-2): GeoJSON/CSV 다운로드 버튼

이제 그 결과를 **파일로 내려받게** 합니다. 영역을 작은 격자(셀)로 나눠 셀별 평균 변화량을 계산하고, **GeoJSON/CSV 다운로드 버튼**을 화면에 답니다.

> [!quote] 4-2
> 이제 이 결과를 파일로 내려받게 하고 싶어. 영역을 사이드바에서 고른 격자 크기(예: 2km)로 나눠서 **각 칸의 평균 변화량**을 계산하고, 칸별 (중심좌표, 변화량) 표를 만들어줘. 그 표를 화면에 pandas로 미리 보여주고, 아래에 **CSV 내려받기 / GeoJSON 내려받기** 두 개의 `st.download_button` 을 달아줘. CSV는 엑셀로, GeoJSON은 QGIS로 바로 열리게 하고, 파일 이름엔 기간이 들어가게 자동으로 지어줘.

격자(셀) 단위로 내보내면 뭐가 좋을까요? → 영역 전체를 한 덩어리 숫자로 뭉개지 않고, **칸칸이 어디가 얼마나 변했는지**를 남깁니다. 그래야 GeoJSON으로 지도에 펴 봤을 때 "이 칸이 가장 많이 줄었다"가 그 자리에 보입니다.

> [!success] 확인 (브라우저)
> 카드 아래에 셀 표가 미리보기로 뜨고, **⬇️ CSV 내려받기 (엑셀)** · **⬇️ GeoJSON 내려받기 (QGIS)** 버튼 두 개가 보입니다. CSV 버튼을 누르면 브라우저가 `change_20230101_to_20240101.csv` 를 내려받습니다. 다운로드 폴더에 그 파일이 떨어지면 성공입니다.

### 단계 3 — 점검 (프롬프트 4-3): 내려받은 파일 값 확인

파일이 내려왔다고 끝이 아닙니다. **그 파일을 다시 열어** 값이 상식과 맞는지 봅니다. 셀 개수가 영역 크기와 어울리는지, 변화량이 터무니없이 크지 않은지(±1을 넘으면 구름·계절 노이즈 의심) 확인합니다. 코드가 만든 숫자를 그대로 믿지 않는 습관입니다. ([[0-first-vibe-coding|할루시네이션 검증]])

> [!quote] 4-3
> 방금 내려받은 CSV를 엑셀로 열었어. 값이 제대로 들어갔는지 같이 확인하고 싶어. 셀이 몇 개고, 변화량(cell_change)의 최소·최대·평균이 얼마면 정상인지 알려줘. 너무 크면(±1 이상) 뭘 의심해야 해? 그리고 화면 미리보기 표와 내려받은 파일의 값이 같은지 어떻게 대조하면 돼?

> [!success] 확인 (브라우저 + 엑셀)
> 내려받은 CSV를 엑셀로 열면 `cell_id, lon, lat, cell_change` 네 열이 보이고, 화면 미리보기 표와 같은 행 수·같은 값입니다. `cell_change` 열을 정렬해 보면 대략 이런 범위입니다.
> ```
> 셀 개수      : 9 개
> 변화량 최소  : -0.214   (가장 많이 감소한 셀)
> 변화량 최대  : +0.157   (가장 많이 증가한 셀)
> 변화량 평균  : -0.018
> ```
> 범위가 대략 **-0.5 ~ +0.5** 면 정상입니다. 가장 많이 감소한 셀의 좌표(lon, lat)가 실제 개발지·벌채지 근처인지 지도로도 대보세요. ([[0-first-vibe-coding|할루시네이션 검증]])

## 막히면

> [!warning] 자주 나는 오류
> - **카드가 안 뜨고 "맑은 영상이 없어요"** → 그 기간에 구름 임계값 미만 영상이 없는 것. **구름 임계값 슬라이더**를 30~40으로 올리거나 기간을 넓히세요. → [[1-prompts|구름 필터]]
> - **변화량이 ±1을 넘게 큼** → 이전/이후가 계절이 다른 영상일 수 있음. 두 기간을 같은 계절(예: 둘 다 여름)로 맞추세요. → [[3-change-detection|변화 해석하기]]
> - **셀이 수백 개로 폭증 / 느림** → 반경이 큰 것. 사이드바 **격자 크기**를 5~10km로 키워 셀 수를 줄이세요.
> - **다운로드 버튼이 안 보임** → [분석 / 보기]를 먼저 눌러 결과가 나와야 버튼이 생깁니다.
> - **`streamlit: command not found`** → `pip install -r requirements.txt` 후 다시.
> - 완성 코드는 `datawa_study04/app.py` 참고.

## 다음

이 장에서 변화탐지가 **구름 임계값까지 사용자가 고르고, 결과를 카드로 보고 파일로 내려받는** 쓸 만한 서비스(`run_change_detection`)가 됐습니다. 한 지역을 손으로 한 번씩 돌린 셈이죠.
