# datawa_study03 · 미니 변화탐지 웹앱

3장의 결과물. **두 시점(이전 vs 이후)의 식생 변화를 색으로 보여주는 Streamlit 웹앱**입니다.
2장(자연색 / NDVI 보기) 위에 **변화탐지**를 더한 **누적 앱**입니다 — 사이드바에서 지역·기간을 정하고 보기 모드(자연색 / NDVI / 변화)를 고르면 브라우저 지도에 바로 그려집니다.

이 장이 **관통 프로젝트의 심장**입니다. 4장부터는 이 심장(`ndvi_change`)을 쓸 만한 서비스로 키웁니다.

## 만든 방법 (뼈대 → 기능 → 점검)

[PROMPTS.md](../PROMPTS.md)의 프롬프트 **3-1 → 3-2 → 3-3** 순서로 Claude Code에게 시켜 단계별로 키웠습니다.

- **3-1 뼈대** — 2장 `ndvi_for_roi` 를 **두 번** 호출해 두 시점 NDVI를 빼는 함수(`ndvi_change`). 색은 아직 안 칠합니다.
- **3-2 기능** — 그 변화량을 빨강~흰~초록 **발산형 히트맵**으로 지도에 올리고, "변화" 보기 모드와 범례를 붙입니다.
- **3-3 점검** — 아는 변화지역(개발지·벌채지)에서 감소(빨강)가 높게 나오는지 값으로 검증합니다.

## 실행

```bash
pip install -r requirements.txt
earthengine authenticate     # 처음 한 번만 (브라우저 로그인)
streamlit run app.py
```

실행하면 **브라우저가 자동으로 열리고** 지도가 보입니다.

## 사용법

1. 왼쪽 사이드바에서 **중심 좌표·반경**으로 분석할 지역을 정한다.
2. **이전 기간 / 이후 기간**을 각각 정한다. (비우면 이전=2023년, 이후=2024년)
3. **보기 모드**에서 `변화` 를 고른다.
4. 지도가 빨강~흰~초록 히트맵으로 칠해지고, 색-값 범례가 함께 뜬다.
5. (점검) 이미 변한 걸 아는 지역(개발지·벌채지)으로 좌표를 옮겨, 감소(빨강)가 진하게 잡히는지 확인한다.

## 주요 함수

- `make_roi(lon, lat, radius_km)` → 중심 좌표·반경으로 분석 영역(`ee.Geometry`)을 만든다. (2장 누적)
- `ndvi_for_roi(roi, start, end)` → 한 시점의 **NDVI 이미지**(`ee.Image`, 밴드 "NDVI")를 반환한다. `normalizedDifference(["B8", "B4"])`. (2장 누적)
- `ndvi_change(roi, before, after)` → **이 장의 심장.** 두 시점 NDVI를 빼서 **변화량 이미지**(밴드 "NDVI_change", = 이후 − 이전)를 반환한다. **4장 서비스가 이 함수를 사용자 입력으로 호출해 재사용합니다.**

## 확인 (이게 보이면 성공)

- 보기 모드 `변화` 에서 영역이 **빨강~흰~초록** 히트맵으로 칠해지고, 범례(colorbar)가 보인다. 🎉
- **변화 없는 넓은 면은 흰색에 가깝게**, 개발·벌채된 곳은 빨강, 식생이 늘어난 곳은 초록으로 뜬다.
- 보기 모드를 `자연색` / `NDVI` 로 바꾸면 2장에서 만든 화면이 그대로 나온다.

## 막히면

- `ee.Initialize` 오류 / `Please authorize...` → 터미널에서 `earthengine authenticate` 를 한 번 실행.
- **온통 흰색** → 변화량이 작아 색이 안 보이는 것. `app.py` 의 `CHANGE_VIS` 에서 `min`/`max` 를 `-0.2`/`0.2` 처럼 좁혀 대비를 키우세요.
- **온통 빨강 또는 초록** → 두 기간의 **계절이 달라** 식생이 통째로 바뀐 것처럼 보이는 가짜 변화. 이전/이후를 같은 계절(예: 둘 다 여름)로 맞추세요.
- **잡티처럼 빨강·초록 점이 너무 많음** → 두 시점 중 하나에 구름·그림자가 낀 것. 구름 임계값을 낮추거나 기간을 넓혀 더 맑은 영상이 들어가게 하세요.
- **개발지인데 증가(초록)가 더 많음** → 이전/이후 순서가 바뀐 것. 변화량은 `after.subtract(before)`(이후 − 이전)가 맞습니다.
- 지도가 안 뜸(빈 화면) → `import geemap.foliumap as geemap` 인지 확인. Streamlit 안에서는 folium 백엔드라야 지도가 임베드됩니다.
- 그 외 에러는 메시지를 복사해 Claude Code에게 물어보세요.

> 코드 검증 메모: NDVI 식(`normalizedDifference(["B8","B4"])`)·발산형 팔레트(`["#d73027","#ffffff","#1a9850"]`, min/max −0.3~0.3)·`subtract` 차분은 참고 프로젝트 `dkidi`·`dkchangedetection` 의 실제 패턴에 근거합니다.
