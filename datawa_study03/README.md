# datawa_study03 · 미니 변화탐지 앱

3장의 결과물. **작년 vs 올해의 변화를 빨강~초록 히트맵으로 보여주는 앱**입니다.
2장에서 "한 시점의 식생을 읽는" 데서 나아가, 두 시점을 **빼서** 식생이 어디서 늘고 줄었는지 — 즉 **변화**를 찾습니다. 관통 프로젝트의 심장입니다.

## 만든 방법 (뼈대 → 기능 → 점검)

[PROMPTS.md](../PROMPTS.md)의 프롬프트 **3-1 → 3-2 → 3-3** 순서로 Claude Code에게 시켜 단계별로 키웠습니다.

- **3-1 뼈대** — 2장의 `ndvi_for_roi` 를 **두 시점**에 호출해 빼는 함수(`ndvi_change`)부터. 색은 아직 안 칠합니다.
- **3-2 기능** — 그 변화량을 빨강(감소)~흰(없음)~초록(증가) **발산형** 색지도로 지도에 올리고 범례를 붙입니다.
- **3-3 점검** — 임계값을 넘는 큰 감소/증가가 각각 몇 %인지 뽑아, 아는 변화지역으로 값을 검증합니다.

## 실행 (주피터/코랩 권장)

```bash
pip install geemap earthengine-api jupyter
jupyter notebook   # 또는 Colab에 change_detect.py 내용을 붙여넣기
```

`change_detect.py` 의 코드를 노트북 셀에 붙여넣고 실행하면 지도가 뜹니다. (1·2장과 같은 환경이면 추가 설치는 필요 없습니다.)

## 사용법

1. 지도 왼쪽의 사각형 그리기 도구로 관심 영역을 그린다. (1·2장과 동일)
2. `show_change(m.user_roi)` 를 실행한다.
3. 영역이 빨강~흰~초록 히트맵으로 칠해지고, 색-값 범례가 함께 뜬다.
4. (점검) `change_stats(m.user_roi)` 로 큰 감소/증가 비율을 숫자로 확인한다.

같은 계절끼리 비교하면 가짜 변화(계절 노이즈)가 줄어듭니다:

```python
show_change(m.user_roi, before=("2020-06-01", "2020-09-30"),
                        after=("2024-06-01", "2024-09-30"))
```

## 주요 함수

- `ndvi_for_roi(roi, start, end)` → **NDVI 이미지(`ee.Image`, 밴드 "NDVI")를 반환** (2장과 동일한 함수). 3장이 자체 포함 스크립트라 같은 파일에 다시 넣었습니다.
- `ndvi_change(roi, before, after)` → **변화량 이미지(`ee.Image`, 밴드 "NDVI_change")를 반환**한다. `after`의 NDVI − `before`의 NDVI. 2장 함수를 두 번 호출해 `subtract` 로 뺍니다. **4장 서비스가 이 함수를 사용자가 고른 기간/지역으로 호출해 재사용합니다.**
- `show_change(roi, ...)` → 위 변화량을 빨강~흰~초록 발산형 색지도+범례로 지도에 올린다.
- `change_stats(roi, ..., threshold=0.2)` → 임계값을 넘는 큰 감소/증가 비율(%)을 출력해 검증.

## 확인 (이게 보이면 성공)

- 그린 영역이 빨강~흰~초록 히트맵으로 칠해지고, 범례(colorbar)가 보인다.
- **변화 없는 곳은 흰색**, 식생이 줄어든 곳(개발·벌채)은 빨강, 늘어난 곳은 초록으로 구분된다.
- `change_stats` 에서 큰 감소/증가 비율이 숫자(%)로 나오고, 아는 변화지역이면 상식과 맞는다.

## 막히면

- `user_roi` 가 `None` → 아직 영역을 안 그렸거나, 그린 뒤 셀을 다시 실행하지 않은 경우.
- **온통 흰색** → `vmax` 가 너무 큰 것. 변화량은 보통 −0.3~0.3 사이. `show_change(m.user_roi, vmax=0.2)` 처럼 줄여 대비를 키우세요.
- **온통 빨강/초록뿐** → 두 기간의 **계절이 달라** 통째로 식생이 바뀐 것처럼 보이는 경우. `before`/`after` 를 같은 계절(예: 둘 다 여름)로 맞추세요.
- 변화가 너무 잡티처럼 많음 → 구름·그림자 노이즈일 수 있음. 두 기간 모두 맑은 영상이 들어가도록 기간을 넓히세요.
- `add_colorbar` 가 없다는 에러 → geemap 버전이 낮은 경우. `pip install -U geemap` 로 올려보세요.
- 그 외 에러는 메시지를 복사해 Claude Code에게 물어보세요. (프롬프트 0-2 패턴)

> 코드 검증 메모: 두 시점 차분(`after.subtract(before)`)과 발산형 팔레트·대칭 min/max(±0.3)는 참고 프로젝트 `dkchangedetection/backend/app/services/imagery/gee_change.py` 의 변화 방향 시각화(`direction` 밴드 palette `["#2166ac","#f7f7f7","#b2182b"]`, loss/none/gain의 발산형 3색 + 대칭 min/max −1~1)와 동일한 정신입니다. 교재는 초급자용으로 "두 시점 NDVI 차분" 단순 버전을 쓰고, 색은 의미가 직관적인 빨강(감소)~흰~초록(증가)으로 바꿨습니다. `reduceRegion` 으로 면적 비율을 뽑는 패턴은 2장 `compare_ndvi` 의 `reduceRegion` 과 같은 방식입니다.
