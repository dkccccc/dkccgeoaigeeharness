# Obsidian Vault (볼트 조립 · LLM 위키 큐레이션)

개별 노트를 하나의 항해 가능한 Obsidian 볼트로 조립한다. 위키링크가 모두 연결되고, MOC에서 어디로든 도달 가능하며, 그래프가 학습 경로를 드러내야 한다. 완성본을 사용자 지정 경로로 내보낸다.

## 조립 절차

1. **폴더 구성**: curriculum-architect의 `01_vault-structure.md`에 따라 폴더 트리를 만들고, `_workspace/02_concepts/`·`_workspace/03_labs/`의 노트를 제자리에 배치한다.
2. **QA 게이트**: tech-validator의 `qa-report.md`에서 심각도 '높음'이 미해결인 노트는 볼트에 넣지 않거나 `> [!warning] 검수중` 표시. 통과분만 정식 배치.
3. **MOC 작성/갱신**: 메인 MOC + 챕터 인덱스. 모든 노트가 최소 1개 MOC에서 도달 가능해야 한다.
4. **링크 무결성 검사**: 모든 `[[...]]`가 실제 노트로 연결되는지, 고아 노트·끊긴 백링크가 없는지 점검하고 고친다.
5. **태그 일관화**: 주제/유형/난이도 태그를 규약대로 적용.
6. **내보내기**: 사용자 지정 경로(미지정 시 프로젝트 루트 `vault/`)로 완성 볼트 출력.

## MOC 템플릿

```markdown
# 00 · 변화탐지 앱으로 배우는 GeoAI (메인 MOC)

> [!info] 이 교재는
> 원격탐사 변화탐지 앱을 직접 만들며 GEE·GeoAI·Claude Code·Harness를 배운다.

## 학습 경로 (순서대로)
1. [[00-시작하기]] — 환경 준비
2. **GEE 기초** → [[01-gee-MOC]]
3. **GeoAI 분석** → [[02-geoai-MOC]]
4. **Claude Code 자동화** → [[03-claude-code-MOC]]
5. **Harness 오케스트레이션** → [[04-harness-MOC]]
6. [[05-마무리]]

## 주제별 개념 모음
- #주제/gee  #주제/geoai  #주제/claude-code  #주제/harness
```

챕터 MOC는 그 챕터의 실습을 순서대로 + 관련 개념 노트를 링크한다.

## 링크 무결성 규칙

- 위키링크 대상은 실제 파일명과 **정확히** 일치(대소문자·공백·한글). Obsidian은 표시 텍스트에 `[[파일명|보이는 글자]]`를 쓴다.
- **깨진 링크**(대상 노트 없음): 매니페스트와 대조해 (a) 파일명 오타면 링크 수정, (b) 노트 누락이면 해당 작성자에게 SendMessage.
- **고아 노트**(들어오는 링크 0): 최소 1개 MOC나 관련 노트에서 링크되도록 연결.
- 노트 **내용은 바꾸지 않는다**. 구조·링크·태그·MOC만. 내용 문제는 작성자에게 돌린다.

## 산출물

- 최종 Obsidian 볼트 → 사용자 지정 경로(또는 `vault/`).
- `_workspace/05_curation/link-audit.md` — 깨진 링크/고아 노트/수정 내역 요약.

## 체크리스트

- [ ] 모든 노트가 규약 폴더에 배치됐는가
- [ ] 메인 MOC + 챕터 MOC가 있고, 모든 노트가 도달 가능한가
- [ ] 깨진 위키링크 0, 고아 노트 0인가
- [ ] 태그가 일관 적용됐는가
- [ ] QA 미통과(심각도 높음) 노트가 정식 배치되지 않았는가
- [ ] 완성 볼트가 지정 경로로 내보내졌는가
