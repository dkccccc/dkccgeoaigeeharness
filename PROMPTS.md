## 1장 · 프롬프트가 전부다 — 위성영상 뷰어

**1-1. 뼈대 만들기**
```
지도에서 사각형 영역을 그리면, 그 영역의 최근 Sentinel-2 영상을 보여주는
간단한 뷰어를 geemap으로 만들고 싶어. 먼저 디자인이나 기능은 신경 쓰지 말고,
'지도를 띄우고 영역을 그릴 수 있는' 최소한의 뼈대부터 만들어줘.
```

**1-2. 기능 붙이기**
```
좋아. 이제 내가 그린 영역에 대해서만 위성영상이 보이도록 기능을 추가해줘.
구름이 많은 영상은 자동으로 걸러줬으면 좋겠어.
```

**1-3. 점검하기 (할루시네이션 확인)**
```
방금 추가한 구름 필터가 실제로 동작하는지 어떻게 확인할 수 있어?
구름 낀 날짜와 맑은 날짜를 비교해서 보여주는 방법을 알려줘.
```

## 2장 · 클로드 코드 실전 — NDVI 식생지도 앱

**2-1. 뼈대 만들기 (NDVI 계산)**
```
좋아, 1장에서 만든 뷰어가 잘 돌아가. 이제 한 가지만 추가하자.
내가 그린 영역에 대해 NDVI(식생지수)를 계산하는 함수를 만들어줘.
구름 적은 Sentinel-2 영상에서 계산했으면 좋겠어. 디자인은 아직 신경 쓰지 말고,
NDVI 이미지를 깔끔하게 반환하는 함수 하나부터.
```

**2-2. 기능 붙이기 (색지도 + 범례)**
```
이제 그 NDVI를 지도에 색으로 보여줘. 식생이 적은 곳은 갈색,
많은 곳은 초록색으로 칠해줬으면 좋겠어. 색이 무슨 값을 뜻하는지
알 수 있게 범례(colorbar)도 같이 띄워줘.
```

**2-3. 점검하기 (값이 상식과 맞나)**
```
NDVI가 제대로 계산된 건지 확인하고 싶어. 영역 안의 물(강이나 호수)과 숲,
두 지점의 NDVI 평균값을 뽑아서 비교해줘.
물이 낮고 숲이 높게 나오면 제대로 된 거지?
```

## 3장 · 변화를 탐지하기 — 미니 변화탐지 앱

**3-1. 뼈대 만들기 (두 시점 NDVI 차분)**
```
좋아, 2장에서 만든 ndvi_for_roi 함수가 잘 돌아가. 이제 그 함수를 그대로 써서
변화탐지를 하자. 같은 영역에 대해 작년(2023)과 올해(2024)의 NDVI를 각각 구한
다음, 둘을 빼서 변화량 이미지를 반환하는 함수 ndvi_change 를 만들어줘. 디자인은
아직 신경 쓰지 말고, "올해 − 작년" 변화량 이미지 하나를 깔끔하게 반환하는
함수부터. 새 영상을 다시 받지 말고 기존 ndvi_for_roi 를 두 번 호출했으면 좋겠어.
```

**3-2. 기능 붙이기 (발산형 히트맵 + 범례)**
```
이제 그 변화량을 지도에 색으로 보여줘. 식생이 줄어든 곳은 빨강, 변화 없는 곳은
흰색, 늘어난 곳은 초록으로 칠하는 발산형 히트맵으로. 0을 가운데 흰색에 두고
양끝을 대칭으로(예: −0.3 ~ +0.3) 잡아줘. 색이 무슨 값을 뜻하는지 알 수 있게
범례(colorbar)도 같이 띄워줘.
```

**3-3. 점검하기 (임계값 통계로 검증)**
```
변화가 진짜인지 확인하고 싶어. 영역 안에서 변화량의 절댓값이 임계값(예: 0.2)보다
큰 픽셀이 몇 %인지, 큰 감소(빨강)와 큰 증가(초록)를 나눠서 출력해줘. 내가
개발지나 벌채지를 그렸으면 감소 비율이 높게 나와야 맞는 거지?
```

## 4장 · 진짜 데이터 붙이기 — 쓸 만한 변화탐지 서비스

**4-1. 뼈대 만들기 (변화탐지를 서비스로 파라미터화)**
```
좋아, 3장 변화탐지가 잘 돌아가. 이제 이걸 '서비스'로 만들고 싶어.
지역, 이전 기간, 이후 기간, 그리고 구름 임계값을 인자로 받는
run_change_detection 함수 하나로 감싸줘. 변화 이미지 하나만 주지 말고,
평균 변화량과 식생이 감소·증가한 면적 비율 같은 요약 통계도 같이 담은
딕셔너리로 돌려줘. 그래야 나중에 여러 지역에 반복해 쓸 수 있어.
```

**4-2. 기능 붙이기 (GeoJSON/CSV 내보내기)**
```
이제 이 결과를 파일로 내보내고 싶어. 영역을 2km 정도 격자로 나눠서
각 칸의 평균 변화량을 계산하고, GeoJSON과 CSV 둘 중에 골라서 저장하는
export_result 함수를 만들어줘. CSV는 엑셀로, GeoJSON은 QGIS로 바로 열 수 있게.
파일 이름은 기간이 들어가게 자동으로 지어줘.
```

**4-3. 점검하기 (내보낸 파일 값 확인)**
```
방금 내보낸 파일을 다시 열어서 값이 제대로 들어갔는지 확인하고 싶어.
셀이 몇 개고, 변화량의 최소·최대·평균이 얼마인지 출력해줘.
변화량이 보통 어느 범위면 정상인지도 알려줘. 너무 크면 뭘 의심해야 해?
```

## 5장 · AI 에이전트 팀 — 여러 지역 자동 분석

**5-1. 뼈대 만들기 (여러 지역 반복)**
```
좋아, 4장 run_change_detection 이 한 지역을 잘 분석해. 이제 한 지역이 아니라
여러 관심지역을 한 번에 돌리고 싶어. 지역마다 이름과 중심 좌표를 적은 목록을
받아서, 각 지역에 run_change_detection 을 차례로 호출하고 결과를 모아 돌려주는
batch_change_detection(aoi_list, before, after, cloud_pct) 함수를 만들어줘.
한 지역에서 맑은 영상이 없어 실패해도 멈추지 말고, 그 지역만 건너뛰고 다음으로
넘어가게 해줘.
```

**5-2. 기능 붙이기 (결과를 보고서로)**
```
이제 batch_change_detection 이 모은 결과를 한 장의 보고서로 묶고 싶어.
지역 이름, 평균 변화량, 식생 감소 %, 식생 증가 %를 칸으로 한 마크다운 표를
만들고, change_report.md 파일로 저장하는 build_report(results) 함수를 만들어줘.
변화가 가장 많이 감소한 지역이 표 맨 위로 오게 정렬하고, 맨 아래에
"가장 주의해서 볼 지역" 한 줄 요약도 붙여줘.
```

**5-3. 점검하기 (보고서 값 교차확인)**
```
방금 만든 보고서의 값들이 상식과 맞는지 점검하고 싶어. 평균 변화량이 보통
어느 범위면 정상이고, 너무 크면 뭘 의심해야 하는지 알려줘. 각 지역의 감소%와
증가% 합이 말이 되는지도 확인하고, 이상한 지역이 있으면 짚어주는
check_report(results) 함수를 만들어줘.
```

## 6장 · 세상에 내보내기 — 배포와 공유

**6-1. 뼈대 만들기 (Streamlit 웹앱)**
```
좋아, 4장에서 만든 run_change_detection 변화탐지 함수가 잘 돌아가. 이제 이걸
동료가 브라우저에서 클릭해 쓰는 웹앱으로 만들고 싶어. Streamlit으로, 왼쪽
사이드바에서 중심 좌표·반경(km)·이전 기간·이후 기간·구름 임계값을 입력받고,
[분석 실행] 버튼을 누르면 run_change_detection을 호출해서 변화 지도와 요약
통계(면적·평균 변화량·감소/증가 비율)를 보여줘. 지도는 geemap을 streamlit
안에 임베드해줘. 코드는 한 파일 app.py로, 한국어 주석으로.
```

**6-2. 기능 붙이기 (배포해 링크 공유)**
```
이 Streamlit 앱을 무료로 배포해서 링크로 공유하고 싶어. Streamlit Community
Cloud에 올리는 절차를 처음부터 알려줘. GitHub에 뭘 올려야 하는지(app.py 말고
requirements.txt도 필요하지?), 그리고 클라우드에는 브라우저 인증 창이 없으니까
Earth Engine 인증을 서비스 계정으로 어떻게 넣는지도 단계별로 설명해줘.
```

**6-3. 점검하기 (MCP 맛보기)**
```
MCP(Model Context Protocol)가 뭔지 초보자 눈높이로 짧게 설명해줘. 그리고 내
변화탐지 결과 파일들이 든 폴더를 Claude Code가 직접 읽을 수 있게, 가장 간단한
MCP 서버(예: filesystem) 하나를 연결하는 방법을 알려줘. 설정 파일에 뭘 적어야
하는지 한 줄씩 짚어주고, 연결이 됐는지 확인하는 법도 알려줘.
```

## 프롬프트 잘 쓰는 4가지 습관

1. **나를 소개한다** — "나는 원격탐사 연구자, 코딩 초보야" → Claude가 눈높이를 맞춘다
2. **한 번에 하나씩** — 뼈대 먼저, 기능은 그다음. 한꺼번에 시키면 어디가 틀렸는지 모른다
3. **왜를 말한다** — "구름을 걸러줘" 보다 "변화를 보려는데 구름이 방해돼서 걸러줘"
4. **결과를 의심한다** — "정말 되는지 확인하는 법도 알려줘"
